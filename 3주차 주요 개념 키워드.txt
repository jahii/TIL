-PR : 내 작업을 다른 사람에게 반영해 주겠니? 라고 제안하는 것. github에서 할 수 있음.
-amend : 가장 최근에 commit한 내역을 고칠 수 있는 명령어. 
-reset : 강제 reset은 위험! 돌이킬 수 없음
-stash : 변경 사항을 임의로 저장(현업에서는 hot fix)
-README.md : markdown 형식의 파일을 만들어 내 프로젝트 소개하기
-gitignore : 프로젝트 파일에 .gitignore를 추가하여 중요한 정보(비밀번호, 개인 주소, API key 등등)를 넣지 않기 위해 목록을 작성. gitignore에 들어가서 각 언어(또는 프로젝트)에 맞는 gitigonre 목록을 불러들일 수 있다.
-오픈 소스 : 모든 사람들이 참여할 수 있고, 활용할 수 있는 프로젝트. 라이센스를 잘 확인해야 함.
